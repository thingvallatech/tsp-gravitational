The function signature:
```typescript
export function generateCities(preset: PresetName, seed: number, count?: number): City[]
```

Default count = 20 (use DEFAULT_CITY_COUNT from types.ts).

Implement all 6 generators:
- **random**: Cities placed uniformly in [0.1, 0.9] range (margin from edges)
- **clustered**: 3-5 cluster centers, cities scattered around them with 0.15 spread, clamped to [0, 1]
- **circular**: Cities evenly spaced on a circle (radius ~0.35) centered at (0.5, 0.5) with slight radial jitter
- **grid**: Cities on a square grid with slight position jitter (0.02)
- **star**: 5-armed star pattern, cities distributed along arms with jitter
- **spiral**: Cities along a 2-rotation spiral from center outward with jitter

Critical rules:
1. Create the PRNG at the TOP of generateCities using `mulberry32(seed)` -- pass the rng closure to each generator function
2. NEVER use `Math.random()` -- all randomness flows through the seeded rng
3. All city coordinates must be in [0, 1] range -- clamp if needed
4. Each city gets a sequential `id` starting from 0
5. Import types from `./types` and mulberry32 from `./prng`

  <name>Task 2: Implement surface computation and Zustand store</name>

Create 4 files:

**src/lib/surface.ts** -- Gravitational height field computation:
```typescript
export function computeHeightField(cities: City[], params: SurfaceParams): SurfaceData
```
Implementation:
1. Create a `Float32Array` of size `gridResolution * gridResolution`
2. Pre-compute isolation weights via `computeIsolationWeights(cities)` from math-utils
3. Compute centroid via `computeCentroid(cities)` from math-utils
4. Pre-compute `invTwoSigmaSq = 1 / (2 * kernelWidth * kernelWidth)` outside the loop (performance)
5. For each grid cell (gx, gy), map to world coords wx, wy in [0, 1]:
   - Lift term: `h = liftAlpha * euclideanDistance(wx, wy, cx, cy)`
   - For each city i: subtract `weights[i] * exp(-dist^2 * invTwoSigmaSq)` (Gaussian kernel)
6. Track minHeight and maxHeight during computation
7. Return `SurfaceData` object with heightField, gridResolution, minHeight, maxHeight
8. Handle edge case: if cities array is empty, return a flat zero surface

Import City, SurfaceParams, SurfaceData from `./types`. Import computeIsolationWeights, computeCentroid, euclideanDistance from `./math-utils`.

**src/store/city-slice.ts** -- City state slice:
Use Zustand v5 `StateCreator` pattern. State:
- `cities: City[]` (initially empty)
- `preset: PresetName` (initially 'random')
- `seed: number` (initially DEFAULT_SEED from types)
- `cityCount: number` (initially DEFAULT_CITY_COUNT from types)
- `setCities: (preset: PresetName, seed: number, count?: number) => void` -- generates cities and triggers surface recomputation by calling `get().recomputeSurface()` after setting cities

The `StoreState` type must be the union of CitySlice & SurfaceSlice. Export it.

**src/store/surface-slice.ts** -- Surface state slice:
State:
- `surfaceData: SurfaceData | null` (initially null)
- `surfaceParams: SurfaceParams` (initially DEFAULT_SURFACE_PARAMS)
- `recomputeSurface: () => void` -- reads cities and surfaceParams from store, calls computeHeightField, sets surfaceData
- `setSurfaceParams: (params: Partial<SurfaceParams>) => void` -- merges params, then calls recomputeSurface

**src/store/index.ts** -- Combined store:
Use Zustand v5 `create<StoreState>()(...)` with the curried form for TypeScript. Spread both slices. Export `useStore`.

Key wiring: `setCities` must call `recomputeSurface()` after updating cities so the surface stays in sync. Use `get()` to access cross-slice actions.

IMPORTANT: No file in `src/store/` should import from `three`, `react-three`, or any rendering library. Store files may import from `react` (for Zustand hooks) and from `src/lib/`.
