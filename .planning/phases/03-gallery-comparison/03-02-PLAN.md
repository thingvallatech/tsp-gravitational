  <name>Task 1: Synchronized playback hook and gallery integration</name>
  <files>src/animation/useSyncPlayback.ts, src/components/GalleryPage.tsx, src/components/AlgoTile.tsx</files>
  <action>
Create useSyncPlayback.ts -- a hook that drives ALL algorithms on the same timeline:
- Uses rAF with delta-time accumulation (same pattern as usePlayback.ts)
- On each tick, reads ALL keys from `algorithmResults` in the store
- For each algorithm, normalizes progress: `normalizedStep = Math.floor((elapsedTicks / maxStepsAcrossAll) * thisAlgo.steps.length)`
  - This means algorithms with fewer steps finish their animation proportionally earlier, but the timeline is shared
  - `maxStepsAcrossAll` = max steps.length across all results
  - Each tick increments a shared `tickCount` (0 to maxStepsAcrossAll)
- Calls `setPlaybackStep(algoId, normalizedStep)` for each algorithm
- When `tickCount >= maxStepsAcrossAll`, set all to their final step and stop
- Exposes: `{ play, pause, reset, isPlaying, progress }` where progress is 0-1 fraction
- Reads `playbackSpeed` from store for tick interval

Update GalleryPage.tsx:
- Add sync playback controls above the grid: Play/Pause button, Reset button, progress bar (simple div with width based on progress percentage), speed slider
- Wire controls to useSyncPlayback hook
- Remove any per-tile playback controls (tiles just read their playbackStep from store and render)

Update AlgoTile.tsx:
- Ensure it reads `playbackStep[algoId]` reactively from store (should already work from 03-01)
- Add a small progress indicator at bottom of tile: a thin bar showing what fraction of steps are complete for this algorithm (step/totalSteps), colored with algo color
