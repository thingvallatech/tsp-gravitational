  <action>
    1. Install dependencies:
       ```
       npm install three @react-three/fiber @react-three/drei d3-contour d3-scale d3-scale-chromatic
       npm install -D @types/d3-contour @types/d3-scale @types/d3-scale-chromatic
       ```
       Note: @types/three is already in devDependencies.

    2. Create `src/visualization/color-scales.ts`:
       - Export a function `heightToColor(normalizedHeight: number): [number, number, number]` that maps [0,1] to RGB using d3-scale-chromatic's interpolateViridis.
       - Export a function `heightToVertexColor(heightField: Float32Array, minHeight: number, maxHeight: number): Float32Array` that returns a flat RGB array (length = heightField.length * 3) for vertex coloring. Use inverted domain: low heights (wells) = dark, high heights (ridges) = bright.

    3. Create `src/visualization/HeightFieldMesh.tsx`:
       - React component using useRef<THREE.Mesh>.
       - Read surfaceData from `useStore(s => s.surfaceData)`.
       - Render a `<mesh>` with `<planeGeometry args={[1, 1, segments, segments]}>` where `segments = gridResolution - 1` (critical: this ensures vertex count matches heightField length exactly).
       - In a useEffect triggered by surfaceData changes:
         - Set each vertex Z position from heightField values: `position.setZ(i, (heightField[i] - minHeight) / range * 0.3)` (0.3 scale factor for visual height).
         - Set vertex colors using the `heightToVertexColor` helper applied to a `<bufferAttribute>` on the geometry's `color` attribute.
         - Call `position.needsUpdate = true` and `geometry.computeVertexNormals()`.
       - Use `<meshStandardMaterial vertexColors side={THREE.DoubleSide} />`.
       - Rotate mesh `rotation={[-Math.PI / 2, 0, 0]}` to lay flat on XZ plane.

    4. Create `src/visualization/CityMarkers3D.tsx`:
       - Read cities from `useStore(s => s.cities)` and surfaceData from `useStore(s => s.surfaceData)`.
       - For each city, render a small sphere (`<sphereGeometry args={[0.012, 16, 16]}>`).
       - Position each sphere at `[city.x - 0.5, elevationFromSurface + 0.01, city.y - 0.5]` where elevation is sampled from the heightField at the nearest grid point (city coords are [0,1], grid is gridResolution x gridResolution). The -0.5 offset centers the mesh since PlaneGeometry is centered at origin.
       - Use a bright color like `#ff4444` so cities stand out against Viridis palette.

    5. Create `src/visualization/HeightFieldScene.tsx`:
       - Mark `'use client'`.
       - Import Canvas from @react-three/fiber, OrbitControls from @react-three/drei.
       - Compose: `<Canvas camera={{ position: [0.5, 0.8, 1.2], fov: 50 }}> <ambientLight /> <directionalLight /> <HeightFieldMesh /> <CityMarkers3D /> <OrbitControls enableDamping dampingFactor={0.1} minDistance={0.3} maxDistance={3} /> </Canvas>`.
       - Style container: `w-full h-[500px]`.

    6. Update `src/app/page.tsx`:
       - Dynamically import HeightFieldScene using `next/dynamic` with `{ ssr: false }` and a loading placeholder.
       - Keep existing CityCanvas and controls. Add the 3D scene above or beside the existing 2D city plot. Add a section header "3D Gravitational Surface".
       - Keep the existing preset selector and seed input working.
  </action>

       - `'use client'` React component with a canvas ref.
       - Read surfaceData from `useStore(s => s.surfaceData)` and cities from `useStore(s => s.cities)`.
       - Canvas size: 500x500 pixels.
       - In useEffect on surfaceData change:
         - Create ImageData at gridResolution x gridResolution.
         - Use `scaleSequential(interpolateViridis).domain([maxHeight, minHeight])` (inverted so wells are dark).
         - Fill pixel RGBA from heightField values. Parse the d3 `rgb(r,g,b)` string output with a regex.
         - Draw to an OffscreenCanvas at grid resolution, then drawImage scaled to canvas size (500x500).
       - After heatmap, draw contour lines:
         - Use `contours().size([gridResolution, gridResolution]).thresholds(15)` on the heightField.
         - Scale context by `canvasSize / gridResolution` before drawing.
         - Stroke each contour polygon in a semi-transparent white (`rgba(255,255,255,0.3)`) with thin lines (`lineWidth: 0.5 / scale`).
       - After contours, draw city dots:
         - For each city, draw a filled circle at `(city.x * canvasSize, city.y * canvasSize)` with radius 5, color `#ff4444`, with a white border.

    2. Update `src/app/page.tsx`:
       - Add the Heatmap2D component below the 3D scene. Add a section header "2D Heatmap".
       - This component does NOT need dynamic import (no Three.js, just Canvas 2D + d3).
       - Layout: stack vertically -- 3D scene on top, 2D heatmap below.

    Note on d3-contour: the heightField indexing `[gy * gridResolution + gx]` is compatible with d3-contour's expected `[i + j * n]` layout. Cast `heightField as unknown as number[]` when passing to the contour generator since d3-contour accepts ArrayLike<number>.
