    1. Create `src/algorithms/types.ts`:
       ```typescript
       export interface AlgoStep {
         tour: number[];           // city indices in visit order (partial or complete)
         edges: [number, number][]; // edges being considered/added this step
         cost: number;             // current tour length
         description: string;      // human-readable narration of this step
       }

       export type AlgorithmCategory = 'construction' | 'improvement' | 'metaheuristic' | 'novel';

       export interface AlgoResult {
         algoId: string;
         steps: AlgoStep[];
         finalTour: number[];
         tourLength: number;
         computeTimeMs: number;
       }

       export interface TSPAlgorithm {
         id: string;
         name: string;
         category: AlgorithmCategory;
         color: string;
         description: string;
         solve: (cities: City[], surfaceData?: SurfaceData) => AlgoStep[];
         requiresSurface?: boolean;
       }
       ```
       Import City and SurfaceData from `@/lib/types`.

    2. Create `src/algorithms/tour-utils.ts`:
       - `tourLength(cities: City[], tour: number[]): number` -- sum of euclidean distances between consecutive tour cities plus closing edge back to start. Use `euclideanDistance` from `@/lib/math-utils`.
       - `tourEdges(tour: number[]): [number, number][]` -- returns edges as pairs including the closing edge.

    3. Create `src/algorithms/nearest-neighbor.ts`:
       - Follow the pattern from research exactly. Start at city index 0, greedily visit nearest unvisited city. Push an AlgoStep for each city added. Final step closes the tour.
       - Each step description: `"Nearest Neighbor: connecting city {id} to city {id} (distance {d.toFixed(3)})"`.

    4. Create `src/algorithms/greedy.ts`:
       - Sort all edges by distance. Add shortest edges that don't create a vertex with degree > 2 and don't close a cycle prematurely (until all cities are connected). Track degree of each vertex.
       - Push AlgoStep for each edge added: `"Greedy: adding edge {a}-{b} (distance {d.toFixed(3)})"`.

    5. Create `src/algorithms/two-opt.ts`:
       - Start with initial tour [0,1,2,...,n-1]. Repeatedly try all (i,j) segment reversals. If reversal reduces tour length, apply it and record a step.
       - Step description: `"2-Opt: reversing segment [{i+1}..{j}], saved {saved.toFixed(3)}"`.
       - Continue until no improvement found in a full pass.

    6. Create `src/algorithms/three-opt.ts`:
       - Start with initial tour [0,1,...,n-1]. Try all (i,j,k) segment reconnections (8 possible reconnections per triple). Accept the best improving reconnection.
       - Record a step for each improvement found.
       - Step description: `"3-Opt: reconnecting segments at positions {i},{j},{k}, saved {saved.toFixed(3)}"`.
       - Limit iterations to prevent excessive steps (max 3 full passes).

    7. Create `src/algorithms/simulated-annealing.ts`:
       - Start with nearest-neighbor initial tour. Use 2-opt swap as the neighborhood move.
       - Parameters: T_start=1.0, T_end=0.001, cooling=0.995, iterations_per_temp=cities.length*2.
       - Accept worse solutions with probability exp(-delta/T).
       - Record a step every 10 iterations (not every single one -- too many steps for animation).
       - Step description: `"SA: T={T.toFixed(4)}, cost={cost.toFixed(3)}, {accepted/rejected} swap"`.
  </action>

    1. Create `src/algorithms/genetic-algorithm.ts`:
       - Population size: 50. Generations: 100. Tournament selection (size 3). Order crossover (OX1). Swap mutation (rate 0.1).
       - Record step per generation: `"GA: Gen {n}, best={bestCost.toFixed(3)}, avg={avgCost.toFixed(3)}"`.

    2. Create `src/algorithms/ant-colony.ts`:
       - Ants: 20. Iterations: 50. Alpha=1, Beta=2, Evaporation=0.5, Q=1.
       - Record step per iteration: `"ACO: Iteration {n}, best ant tour={bestCost.toFixed(3)}"`.

    3. Create `src/algorithms/christofides-like.ts`:
       - Implement MST-based approximation (not true Christofides which requires perfect matching):
         a. Compute MST using Prim's algorithm. Record step: `"MST: adding edge {a}-{b}"`.
         b. Find vertices with odd degree.
         c. Greedy match odd-degree vertices (nearest unmatched).
         d. Create Eulerian circuit from MST + matching edges.
         e. Shortcut repeated vertices to get Hamiltonian tour.
       - Label honestly: name="MST Approximation", description="MST + greedy matching + shortcutting".

    4. Create `src/algorithms/elastic-net.ts`:
       - Durbin & Willshaw (1987) approach. Initialize ring of M=2.5*N points around centroid.
       - Parameters: alpha=0.2, beta=2.0, K_start=0.2, K_end=0.01, iterations=200.
       - Each iteration: for each city find nearest net node, update net positions toward cities (alpha term) and toward neighbors (beta term), reduce K.
       - Record step every 5 iterations: `"Elastic Net: iteration {n}, K={K.toFixed(4)}, ring deformation"`. Show current net node order mapped to nearest cities as the tour.

    5. Create `src/algorithms/gravitational.ts`:
       - This is the novel algorithm. Takes `surfaceData` parameter (requiresSurface: true).
       - Implementation:
         a. Start at the grid point with maximum height (centroid lift peak).
         b. Simulate steepest-descent on the heightfield: from current position, check 8 neighbors, move to lowest.
         c. When descent reaches a local minimum near a city (within 2 grid cells of any unvisited city), add that city to tour.
         d. After visiting a city's well, "fill" it: set heightField values within kernelWidth radius to the local maximum, then resume descent from that filled point.
         e. Repeat until all cities visited.
       - IMPORTANT: Work on a COPY of the heightField (do not mutate the store's data).
       - Step description: `"Gravitational: drainage reached city {id} at depth {depth.toFixed(3)}"`.
       - If descent gets stuck (revisits same cell), do a random restart from the highest unvisited region.

    6. Create `src/algorithms/registry.ts`:
