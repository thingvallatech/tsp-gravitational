  <action>
    1. Create `src/animation/usePlayback.ts`:
       - Custom hook that drives the animation loop via requestAnimationFrame.
       - Uses delta-time accumulation (NOT frame counting) to advance steps at a consistent rate regardless of monitor refresh rate.
       - Reads `isPlaying`, `playbackSpeed`, `playbackStep`, and `algorithmResults` from the store.
       - Accepts `algoId: string` parameter.
       - On each rAF tick: accumulate deltaMs. While accumulated >= playbackSpeed, call `setPlaybackStep(algoId, current + 1)` and subtract playbackSpeed from accumulated.
       - When step reaches the last step, call `setIsPlaying(false)`.
       - Clean up rAF on unmount or when isPlaying becomes false.
       - Returns `{ currentStep, totalSteps, isPlaying, play, pause, stepForward, stepBackward, reset }` where play/pause/step functions update the store directly.

    2. Create `src/visualization/TourOverlay2D.tsx`:
       - `'use client'` component. Receives `canvasSize: number` prop (same as Heatmap2D, default 500).
       - Renders an absolutely-positioned canvas overlaid on top of the Heatmap2D canvas (use a shared parent div with relative positioning).
       - Reads the currently active algorithm result and current playback step from the store.
       - On each step change, clears and redraws:
         - The tour edges from `steps[currentStep].tour` as connected line segments in the algorithm's color (look up from ALGORITHM_REGISTRY).
         - Line width: 2px. Use `city.x * canvasSize` and `city.y * canvasSize` for coordinates.
         - Draw the closing edge (last city back to first) only when the tour is complete (tour.length === cities.length).
         - Highlight the edges from `steps[currentStep].edges` with thicker lines (3px) to show what was just added/changed.
       - Also draw city dots on top (same as Heatmap2D but on this overlay layer so they appear above tour lines).

    3. Create `src/visualization/TourPath3D.tsx`:
       - `'use client'` component rendered inside the R3F Canvas (in HeightFieldScene).
       - Reads current algorithm result, current step, cities, and surfaceData from the store.
       - Creates a `<line>` using Three.js BufferGeometry with positions sampled from the tour order.
       - For each city in the current step's tour, compute 3D position: `[city.x - 0.5, elevation * 0.3 + 0.005, city.y - 0.5]` where elevation is sampled from heightField (same as CityMarkers3D). The +0.005 lifts the line slightly above the surface.
       - Set line color from the algorithm's color in ALGORITHM_REGISTRY.
       - Use `<lineBasicMaterial linewidth={2} color={algoColor} />`.
       - Update geometry when step changes (set position attribute and call needsUpdate).

    4. Create `src/components/AlgorithmPanel.tsx`:
       - `'use client'` component.
       - Import ALGORITHM_REGISTRY and runAlgorithm.
       - Render a `<select>` dropdown listing all algorithms grouped by category (construction, improvement, metaheuristic, novel). Show algorithm name and a colored dot matching its color.
       - Render a "Run" button. On click:
         a. Get selected algoId.
         b. Read cities and surfaceData from store.
         c. Call `runAlgorithm(algoId, cities, surfaceData)`.
         d. Store result via `setAlgorithmResult(algoId, result)`.
         e. Reset playback to step 0 via `setPlaybackStep(algoId, 0)`.
         f. Set a `selectedAlgoId` state (local or in store) so other components know which algorithm to display.
       - Show the algorithm's description text below the dropdown.
       - Display computation time and final tour length after run completes.

    5. Create `src/components/PlaybackControls.tsx`:
       - `'use client'` component.
       - Uses the `usePlayback` hook with the currently selected algoId.
       - Renders:
         - Play/Pause toggle button (play icon when paused, pause icon when playing). Use simple Unicode characters or text labels.
         - Step backward button (disabled at step 0).
         - Step forward button (disabled at last step).
         - Step counter: "Step {current} / {total}".
         - Speed slider: range from 50ms (fast) to 1000ms (slow) per step, default 300ms. Updates `setPlaybackSpeed`.
       - Disable all controls when no algorithm result is loaded.

    6. Create `src/components/StepNarration.tsx`:
       - `'use client'` component.
       - Reads current algorithm result and current playback step from the store.
       - Displays `steps[currentStep].description` in a styled text box.
       - Shows the current cost: `"Tour length: {cost.toFixed(3)}"`.
       - If no result loaded, show placeholder text: "Select an algorithm and click Run".
  </action>
